# 요구사항 정의서 (PRD - Product Requirements Document)

## 1. 프로젝트 개요
- **제품명**: Stockbot (주식 및 정보 모니터링 텔레그램 봇)
- **목표**: 사용자의 주식 계좌(키움증권) 상태를 모니터링하고 요약하여 텔레그램으로 편리하게 전송하며, 부가적으로 검색 트렌드 및 관련 뉴스를 실시간으로 모니터링하여 종합적인 투자 정보 대시보드 역할을 수행하는 자동화 시스템 구축.

## 2. 주요 기능 요구사항

### 2.1. 키움증권 계좌 연동 및 잔고 조회
- **다중 계좌 지원**: 여러 개의 계좌번호와 각 계좌의 인증키(App Key, Secret Key)를 별도의 파일(`kiwoom_keys/`)로 관리하고 동시 조회를 지원해야 합니다.
- **수동 잔고 조회**: 사용자가 텔레그램 봇에 `/잔고` 명령어를 입력하면 즉시 모든 계좌의 예수금 및 보유 종목 평가 현황을 조회하여 가독성 높은 메시지로 응답해야 합니다.
- **자동 잔고 요약 발송**: 매 평일(월~금) 장 마감 후인 15시 40분에 계좌 상태 요약 메시지를 자동으로 전송해야 합니다.
- **인증 토큰 자동 갱신**: 매일 새벽 3시에 등록된 모든 계좌의 키움증권 API 접근 토큰(Access Token)을 자동으로 갱신해야 합니다.
- **장 점검 시간 대응**: 야간 또는 주말 장 점검 시간 등에 API 응답이 없거나 통신에 실패할 경우 봇이 다운되지 않고 오류 메시지(장 점검 의심 등)를 반환해야 합니다.

### 2.2. 외부 정보 모니터링 (예정/구현 중)
- **구글 트렌드 분석**: 일 단위로 구글 트렌드 데이터를 수집하고, `matplotlib`을 이용해 시계열 차트 이미지로 렌더링하여 텔레그램으로 발송해야 합니다.
- **실시간 뉴스 모니터링**: `config.yaml`에 정의된 특정 키워드(예: "삼성전자", "애플")에 대한 뉴스를 모니터링하고 신규 기사 발생 시 알림을 전송해야 합니다.
- **텔레그램 타 방 모니터링**: User-Bot 기능(`pyrogram` 등 활용)을 통해 특정 텔레그램 채널이나 그룹의 메시지를 감시하고 필요한 정보를 수집해야 합니다.

## 3. 비기능적 요구사항
- **언어 및 문서화**: 프로젝트의 모든 개발 언어는 Python이며, 공식 문서, README 및 코드 내 주석은 반드시 **한글**로 작성해야 합니다.
- **보안**: API Key, Telegram Bot Token 등 어떠한 민감 정보도 소스 코드 내에 하드코딩해서는 안 되며 외부 설정 파일(`config.yaml`, 인증키 파일 등)을 통해 시스템에 주입해야 합니다.
- **서드파티 라이브러리 제약**: 텔레그램 연동 시 반드시 `python-telegram-bot` 패키지를 사용해야 합니다.
- **안정성 및 확장성**: 각 기능(API 호출, 뉴스, 트렌드 등)은 독립된 Task 클래스로 분리되어야 하며, `TaskManager`를 통해 비동기로 스케줄링되어 서로의 작업에 영향을 주지 않아야 합니다. 키움 API 호출 제한(Rate Limit)을 준수하는 재시도 로직을 포함해야 합니다.
