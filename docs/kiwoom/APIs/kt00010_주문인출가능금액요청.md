# 주문인출가능금액요청 (kt00010)

## 1. 기본 정보
- **메뉴 위치**: 국내주식 > 계좌 > 주문인출가능금액요청(kt00010)
- **API 명**: 주문인출가능금액요청
- **API ID**: kt00010
- **Method**: POST
- **운영 도메인**: `https://api.kiwoom.com`
- **모의투자 도메인**: `https://mockapi.kiwoom.com(KRX만 지원가능)`
- **URL**: `/api/dostk/acnt`
- **Format**: JSON
- **Content-Type**: `application/json;charset=UTF-8`

## 2. Request (요청)

### Header
- **api-id** (String, 10, 필수): TR명
- **authorization** (String, 1000, 필수): 접근토큰 (토큰 지정시 토큰타입("Bearer") 붙혀서 호출 
 예) Bearer Egicyx...)
- **cont-yn** (String, 1, 선택): 연속조회여부 (응답 Header의 연속조회여부값이 Y일 경우 다음데이터 요청시 응답 Header의 cont-yn값 세팅)
- **next-key** (String, 50, 선택): 연속조회키 (응답 Header의 연속조회여부값이 Y일 경우 다음데이터 요청시 응답 Header의 next-key값 세팅)

### Body
- **io_amt** (String, 12, 선택): 입출금액
- **stk_cd** (String, 12, 필수): 종목번호
- **trde_tp** (String, 1, 필수): 매매구분 (1:매도, 2:매수)
- **trde_qty** (String, 10, 선택): 매매수량
- **uv** (String, 10, 필수): 매수가격
- **exp_buy_unp** (String, 10, 선택): 예상매수단가

## 3. Response (응답)

### Header
- **api-id** (String, 10, 필수): TR명
- **cont-yn** (String, 1, 선택): 연속조회여부 (다음 데이터가 있을시 Y값 전달)
- **next-key** (String, 50, 선택): 연속조회키 (다음 데이터가 있을시 다음 키값 전달)

### Body
- **profa_20ord_alow_amt** (String, 12, 선택): 증거금20%주문가능금액

- **profa_30ord_alow_amt** (String, 12, 선택): 증거금30%주문가능금액
- **profa_30ord_alowq** (String, 10, 선택): 증거금30%주문가능수량
- **profa_40ord_alow_amt** (String, 12, 선택): 증거금40%주문가능금액
- **profa_40ord_alowq** (String, 10, 선택): 증거금40%주문가능수량
- **profa_50ord_alow_amt** (String, 12, 선택): 증거금50%주문가능금액
- **profa_50ord_alowq** (String, 10, 선택): 증거금50%주문가능수량
- **profa_60ord_alow_amt** (String, 12, 선택): 증거금60%주문가능금액
- **profa_60ord_alowq** (String, 10, 선택): 증거금60%주문가능수량
- **profa_rdex_60ord_alow_amt** (String, 12, 선택): 증거금감면60%주문가능금
- **profa_rdex_60ord_alowq** (String, 10, 선택): 증거금감면60%주문가능수
- **profa_100ord_alow_amt** (String, 12, 선택): 증거금100%주문가능금액
- **profa_100ord_alowq** (String, 10, 선택): 증거금100%주문가능수량
- **pred_reu_alowa** (String, 12, 선택): 전일재사용가능금액
- **tdy_reu_alowa** (String, 12, 선택): 금일재사용가능금액
