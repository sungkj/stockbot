# 아키텍처 결정 기록 (ADR - Architecture Decision Records)

이 문서는 Stockbot 프로젝트 개발 과정에서 내려진 주요 아키텍처 및 기술 스택 관련 의사결정 내역을 기록합니다.

---

## ADR 001: 텔레그램 봇 라이브러리로 `python-telegram-bot` 채택

- **상태**: 승인됨 (Approved)
- **배경**: 파이썬 환경에서 텔레그램 봇을 개발하기 위한 다양한 라이브러리(`pyTelegramBotAPI`, `aiogram`, `python-telegram-bot` 등)가 존재합니다. 프로젝트의 지침에 따라 표준 라이브러리를 하나로 통일할 필요가 있었습니다.
- **결정**: `python-telegram-bot` (PTB) v20 이상을 채택합니다.
- **이유**:
  - 프로젝트 공통 지침(`GEMINI.md`)에서 사용을 명시적으로 요구하고 있습니다.
  - 내장된 비동기 처리(`asyncio`)와 `JobQueue` 기능을 제공하여, 메시지 처리와 주기적인 백그라운드 스케줄링 작업을 하나의 프레임워크 내에서 효율적으로 관리할 수 있습니다.
- **영향**: 봇 앱 초기화 및 핸들러 등록은 PTB의 `ApplicationBuilder` 패턴을 따르며, 스케줄링은 별도 라이브러리 설치 없이 PTB 내장 `job_queue`를 활용합니다.

## ADR 002: 키움 API 통신을 위한 비동기 HTTP 클라이언트 (`aiohttp`) 사용

- **상태**: 승인됨 (Approved)
- **배경**: 키움증권 REST API는 다수의 계좌 정보를 호출해야 하며, 동기식 `requests` 라이브러리를 사용할 경우 응답을 기다리는 동안 봇의 다른 기능(명령어 처리 등)이 블로킹될 수 있습니다.
- **결정**: 키움증권 API 호출을 포함한 모든 외부 네트워크 요청에 `aiohttp`를 사용합니다.
- **이유**:
  - 비동기 IO를 지원하여 여러 계좌의 인증 토큰 발급 및 잔고 조회를 `asyncio.gather`를 통해 병렬로 처리할 수 있어 성능상 유리합니다.
  - 메인 스레드의 이벤트 루프를 블로킹하지 않으므로 봇의 반응성을 유지할 수 있습니다.
- **영향**: 모든 API 호출 함수(`get_deposit_info`, `get_balance_info` 등)는 `async def`로 선언되며 `await` 키워드를 사용하여 호출해야 합니다.

## ADR 003: 키움 API 호출 제한(Rate Limit) 방어 로직 구현

- **상태**: 승인됨 (Approved)
- **배경**: 키움증권 REST API는 단시간 내 과도한 요청 시 '1700' 에러 코드와 함께 호출을 제한(Rate Limit)합니다.
- **결정**: API 호출 래퍼(`fetch_api`)에서 응답의 `err_code` 또는 `return_code`가 '1700'인 경우, `asyncio.sleep`을 이용한 백오프(Back-off) 대기 후 자동으로 재시도하는 로직을 구현합니다.
- **이유**: 
  - 다수 계좌를 순차적 혹은 병렬로 조회할 때 발생할 수 있는 일시적 차단 문제를 시스템 내부적으로 우회하여 안정적인 데이터 수집을 보장하기 위함입니다.
- **영향**: 호출 제한 발생 시 전체 조회 시간이 다소 늘어날 수 있으나 시스템 중단은 발생하지 않습니다.

## ADR 004: 민감 정보의 파일 기반 분리 보관

- **상태**: 승인됨 (Approved)
- **배경**: 프로젝트 지침 상 코드 내부에 API 키 및 계좌번호 하드코딩이 엄격히 금지되어 있습니다. 다중 계좌 지원 구조를 유연하게 가져갈 필요성이 있습니다.
- **결정**: 봇 토큰 등 단일 설정은 `config.yaml`에 보관하고, 여러 개가 될 수 있는 계좌별 키움 API 인증키는 `kiwoom_keys/` 디렉토리 하위에 `[계좌번호]_appkey.txt`, `[계좌번호]_secretkey.txt` 형태로 관리합니다.
- **이유**:
  - 설정 파일의 복잡성을 줄이고, 새로운 계좌 추가 시 소스코드나 설정 파일 수정 없이 디렉토리에 텍스트 파일만 추가함으로써 동적으로 계좌가 로드되는 유연성을 확보합니다.
- **영향**: 애플리케이션 시작 시 `kiwoom_keys/` 디렉토리의 파일들을 스캔하여 메모리에 딕셔너리 형태로 자격 증명을 적재합니다.
