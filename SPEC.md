# 주식 및 종합 정보 제공 텔레그램 봇 프로젝트 기획서

## 1. 프로젝트 개요
`python-telegram-bot`을 기반으로 동작하며, 주식, 뉴스, 개인 계좌 상태, 타 텔레그램 채널의 정보, 구글 트렌드 등 다양한 정보를 수집 및 가공하여 사용자에게 제공하는 텔레그램 봇.

## 2. 시스템 아키텍처 및 핵심 설계 방향
- **모듈화 및 플러그인 구조**: 제공하는 모든 정보(뉴스, 트렌드, 계좌 정보 등)를 독립된 'Task(또는 Job)' 단위로 분리하여 리스트 형태로 관리합니다. 이를 통해 새로운 정보 제공 기능의 추가, 수정, 삭제가 메인 로직에 영향을 주지 않도록 설계합니다.
- **비동기 처리**: 실시간 모니터링, API 호출, 텔레그램 메시지 발송 등의 작업이 병렬적으로 원활하게 수행되도록 비동기(Asyncio) 기반으로 구성합니다.
- **이중 클라이언트 구조**: 봇 자체의 기능(`python-telegram-bot`)과 타 텔레그램 방 모니터링 기능(유저봇 기능, 예: `Telethon` 또는 `Pyrogram`)을 통합하여 운영합니다.

## 3. 기능별 상세 설계

### 3.1 정보 관리 모듈 (Task List Manager)
- **구조**: 알림을 제공할 각 정보(뉴스, 트렌드, 계좌 등)를 하나의 객체(클래스 또는 딕셔너리)로 정의하고, 이를 리스트에 담아 중앙 관리자(Manager)가 순회하며 실행하는 구조입니다. 복잡한 작업의 경우에는 `tasks` 디렉토리 내에 커스텀 클래스를 생성하여 세부 로직을 직접 작성하고 관리할 수 있도록 지원합니다.
- **포함 데이터**:
  - `Task 이름` (예: "삼성전자 실시간 뉴스", "구글 트렌드 일일 보고")
  - `실행 타입` (스케줄링 / 실시간 모니터링)
  - `실행 시각` (스케줄링 타입일 경우, 예: "08:30", "15:30")
  - `트리거 조건` (실시간 타입일 경우, 예: 특정 키워드 매칭)
  - `데이터 수집 함수` (데이터를 가져오는 로직)
  - `메시지 포맷팅 함수` (데이터를 텔레그램 메시지로 예쁘게 꾸미는 로직)
- **장점**: 이 리스트만 수정(설정 파일 분리 가능)하면 봇의 재시작 없이도 알림 항목을 쉽게 켜고 끌 수 있습니다.

### 3.2 스케줄러 기능 (정해진 시각 알림)
- 정보 관리 리스트에 등록된 `실행 시각`을 스케줄러(예: `APScheduler` 또는 내장 JobQueue)가 읽어들여 정해진 시간에 해당 Task의 데이터 수집 및 발송 함수를 호출합니다.

### 3.3 실시간 뉴스 및 웹 모니터링
- 일정 주기(예: 10초~1분)마다 특정 뉴스 포털 RSS나 웹페이지를 스크래핑/API 호출하여 최신 글을 가져옵니다.
- 이전 데이터와 비교하여 '신규 기사' 중 설정된 '조건(키워드 등)'에 부합하는 경우에만 즉시 텔레그램으로 알림을 발송합니다.

### 3.4 키움증권 API 연동 (내 계좌 정보)
- **접근 방식**: 키움증권 REST API를 활용하여 연동합니다.
- **제공 정보**: 잔고 조회, 실시간 미체결 내역, 수익률 달성 알림 등.

### 3.5 타 텔레그램 방 모니터링 (User-Bot)
- `python-telegram-bot`은 봇 계정용이므로, 타 채널/그룹방의 메시지를 읽기 위해서는 사용자의 계정 권한을 빌려오는 User-Bot 라이브러리를 서브 프로세스로 구동합니다.
- 모니터링 중인 방에서 특정 키워드가 포함된 글이나 조건에 맞는 메시지가 감지되면, 메인 봇을 통해 내 텔레그램 방으로 전달(Forwarding/Copy)합니다.

### 3.6 구글 트렌드 분석 및 차트 생성
- 구글 트렌드 비공식 API(`pytrends` 등)를 사용하여 지정된 키워드의 트렌드 수치를 수집합니다.
- 수집된 시계열 데이터를 시각화 라이브러리(`matplotlib`, `plotly` 등)를 사용하여 차트 이미지 파일로 생성합니다.
- 생성된 이미지를 텍스트 분석 결과와 함께 텔레그램 사진 전송 기능으로 발송합니다.

project_root/
│
├── main.py                 # 프로그램 진입점 및 봇 실행
├── config.yaml             # API 키, 알림 시각, 모니터링 키워드 등 환경 설정
├── requirements.txt        # 필요 라이브러리 목록
│
├── core/                   # 봇 핵심 로직
│   ├── bot_app.py          # python-telegram-bot 초기화 및 핸들러 등록
│   └── task_manager.py     # 정보 리스트 관리 및 스케줄러 등록
│
├── tasks/                  # 개별 정보 수집 및 가공 모듈 (플러그인 형태)
│   ├── __init__.py
│   ├── news_monitor.py     # 실시간 뉴스 모니터링 로직

│   ├── google_trends.py    # 구글 트렌드 수집 및 차트 생성 로직
│   └── telegram_monitor.py # 타 텔레그램 방 모니터링 로직 (User-Bot)
│
└── utils/                  # 공통 유틸리티
    ├── logger.py           # 로그 기록 설정
    └── chart_drawer.py     # 차트 이미지 생성 공통 함수
