# 프로젝트 지침 (Stockbot)

본 프로젝트(Stockbot) 작업 시 반드시 준수해야 하는 고유 지침입니다. 여기서는 본 프로젝트에 한정된 규칙만 정의합니다.

## 1. 프로젝트 기획 문서 참조 원칙
- 프로젝트 관련 기능 개발, 구조 변경, 또는 의사 결정을 진행할 때는 반드시 `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/ADR.md` 파일들을 먼저 읽고 해당 문서의 내용을 참고하여 진행해야 합니다.
- 코드 수정 시 PRD.md, ARCHITECTURE.md, ADR.md 파일도 필요한 부분에 한해 업데이트 합니다.

## 2. 보안 및 설정 정보 관리
- **절대** 소스 코드 내에 API Key, Telegram Bot Token, 계좌 번호, 비밀번호 등 개인 정보 및 민감 정보를 하드코딩하지 마십시오.
- 모든 설정 및 민감 정보는 외부 설정 파일(예: `.env`, `config.yaml` 등)을 통해 관리하고 시스템에 주입되도록 구성해야 합니다.

## 3. 키움증권 API 사용 원칙 (가장 중요)
- 키움 API 연동과 관련된 모든 기능(계좌 조회 등) 개발 시, 웹 검색 결과나 기존에 학습된 외부 지식을 절대 사용해서는 안 됩니다.
- 반드시 로컬의 `kiwoom_api_guide.md` 파일의 내용을 최우선으로 참고하고 해당 명세에 맞게 구현해야 합니다.

## 4. 언어 및 라이브러리 사용 제한
- **프로그래밍 언어:** Python을 사용합니다.
- **텔레그램 연동 라이브러리:** 반드시 `python-telegram-bot` 패키지를 사용합니다.
- **문서화 및 언어:** 프로젝트 내의 모든 문서, README, 코드 내 주석은 반드시 **한글**로 작성해야 합니다.
